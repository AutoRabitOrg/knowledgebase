# 1.0.0.202006301216

## Release - 1.0.0.202006301216

### Release Notes:

1. IZ Analyzer Scanner errors out while scanning a non mule projects.
   1. Analyzer plugin will be triggered only for Mule projects with this update
2. Unable to add rules to MUnit Configuration files
   1. MUnit configuration file components can now be accessed using MUNIT\_DOCUMENTS and MUNIT\_COMPONENTS similar to Mule configuration files
3.  Custom Rule - Show errors on a particular file instead of PROJECT\_HANDLE

    1. Errors can be shown on a particular file from custom rules as shown below

    ```groovy
    def results = [:] 
    def scanResult = [:] 
    results.put('FILE', scanResult) 
    String[] ext = ['xml'] 
    Collection srcXmlFiles = org.apache.commons.io.FileUtils.listFiles(new File(ANALYSIS.projectDir, 'src/main/mule'), ext, true) 
    // Change the pattern as per org standards
    def regEx = java.util.regex.Pattern.compile('^[a-z_]+.xml$') 
    srcXmlFiles.each { 
    	if(!regEx.matcher(it.getName()).matches() ) {  
    		// Add the file path and line number. 
    		scanResult.put(it.getAbsolutePath(), 1)  
    	}  
    } 
    return results
    ```
